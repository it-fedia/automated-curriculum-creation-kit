# automated-curriculum-creation-kit

Веб-прототип системы автоматизации сопоставления учебной нагрузки преподавателей и расписания занятий с использованием методов машинного обучения.

## Описание проекта

Проект предназначен для автоматического сопоставления двух источников данных:

- файла учебной нагрузки (РУН);
- файла расписания учебных занятий.

На выходе система формирует:

- расписание по преподавателям в формате Excel;
- список несопоставленных занятий;
- диагностические файлы для анализа качества сопоставления;
- промежуточные результаты для последующей донастройки правил.

Проект реализует гибридный подход к сопоставлению дисциплин:

1. **точное сопоставление** по группе, дисциплине и виду занятия;
2. **нечёткое сопоставление** на основе коэффициента Жаккара;
3. **ML-сопоставление** на основе **TF-IDF + cosine similarity**;
4. **ручную корректировку** через правила маппинга преподавателей.

Такой подход позволяет снизить объём ручной обработки и повысить точность сопоставления в условиях неоднородных Excel-данных.

---

В рамках данного прототипа реализованы:

- обработка табличных Excel-данных;
- нормализация текстовых полей;
- алгоритм сопоставления дисциплин;
- веб-интерфейс для загрузки файлов и получения результата;
- экспорт итогового расписания.

## Ключевой функционал

### Реализовано

- загрузка двух входных Excel-файлов через веб-интерфейс;
- создание рабочей сессии;
- парсинг файла нагрузки;
- парсинг файла расписания;
- нормализация:
  - дней недели;
  - времени;
  - аудиторий;
  - названий дисциплин;
  - видов занятий;
- автоматическое сопоставление преподавателей к занятиям;
- резервный ML-механизм сопоставления дисциплин;
- формирование итогового Excel-расписания по преподавателям;
- формирование списка несопоставленных слотов;
- подбор кандидатов-преподавателей для несопоставленного занятия;
- сохранение пользовательских правил сопоставления;
- повторная пересборка результата с учётом новых правил;
- базовая диагностика конфликтов и low-confidence совпадений;
- очистка устаревших пользовательских сессий.

### Пока не реализовано полностью

- постоянное хранение пользовательских сессий и истории правок;
- полноценная авторизация пользователей;
- расширенное логирование действий через UI;
- автоматическая валидация формата входных Excel-файлов по шаблонам;
- модуль аналитики качества на большом наборе тестовых данных;
- контейнеризация и production-деплой.

## Установка и запуск

### 1. Клонирование репозитория

```bash
git clone https://github.com/it-fedia/automated-curriculum-creation-kit.git
cd automated-curriculum-creation-kit
```
### 2. Создание виртуального окружения
Windows
```bash
python -m venv venv
venv\Scripts\activate
```
Linux / macOS
```bash
python3 -m venv venv
source venv/bin/activate
```

### 3. Установка зависимостей
```bash
pip install -r requirements.txt
```
### 4. Запуск приложения
```bash
python app.py
```

### Как пользоваться системой

1. Открыть веб-интерфейс  
2. Создать новую сессию  
3. Загрузить:
   - файл нагрузки (`run`)
   - файл расписания (`sched`)  
4. Нажать **"Собрать черновик"**  
5. Посмотреть:
   - статистику
   - unmatched-слоты  
6. При необходимости:
   - выбрать кандидата преподавателя
   - сохранить правило  
7. Нажать **"Пересобрать"**  
8. Скачать итоговое расписание  

---

### Алгоритм работы

1. Чтение Excel-файлов  

2. Нормализация:
   - дисциплин
   - видов занятий
   - времени  

3. Сопоставление:

- **Уровень 1 — точное совпадение**  
  группа + дисциплина + тип занятия  

- **Уровень 2 — Jaccard**  
  сравнение токенов дисциплины  

- **Уровень 3 — ML (TF-IDF)**  
  косинусное сходство  

- **Уровень 4 — пользовательские правила**

### API (основные методы)

| Метод | Описание |
|------|--------|
| POST `/api/session/new` | создать сессию |
| POST `/api/upload` | загрузка файла |
| POST `/api/build_draft` | построить черновик |
| GET `/api/unmatched` | получить unmatched |
| GET `/api/candidates` | кандидаты |
| POST `/api/mapping` | сохранить правило |
| POST `/api/rebuild` | пересборка |
| GET `/download/timetable` | скачать результат |
